    Introduction

    Welcome to the Flowback API! You can see all the available endpoints to the API.

    We currently only have language bindings in Shell. You can view code examples in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right.

    Authentication

    To authorize, use this code:

    curl "api_endpoint_here" \
-H "Authorization: Token your_token_here"

    Make sure to replace your_token_here with your API key.

    Flowback uses API keys to allow access to the API. You can get an API key using the Login API endpoint.

    Flowback expects for the API key to be included in all API requests to the server (aside from registration) in a header that looks like the following:

    Authorization: Token your_token_here

    You must replace your_token_here with your personal API key.

    Pagination

    All of the list API endpoints offer pagination support. This feature can be accessed using the query params of the associated API endpoint.

    Every List API will have these parameters in its outermost json object:

    {
  "count": 19,
  "next": "https://v2.flowback.org/api_endpoint?page=2&limit=1&offset=0",
  "previous": false,
  "total_page": 2,
  ...
}

    Query Parameters

    Parameter Default Description
    page int, optional 1 The page of the paginator.
    limit int, optional Varies The maximum amount of entries allowed per page.
    offset int, optional 0 The offset of the paginator, eg. an offset of 2 means the page starts 2 entries ahead per page.

    Login

    Login user using their credentials, returns their auth token

    HTTP Request

    POST https://v2.flowback.org/login

    Body Parameters

    Parameter Notes
    username str The user email
    password str The user password

    Response JSON Parameter

    Parameter Notes
    token The user auth token

    Register

    This endpoint registers an user to the site. An activation email will be sent to the user containing the verification_code. If FLOWBACK_URL is specified in the backend .env, a link containing the verification_code as a query parameter will be sent instead. The code will be needed for the next step of the registration.

    Example of the link: https://v2.flowback.org/register/

    If FLOWBACK_URL is specified in the backend's .env file, the user will instead receive a link:

    curl "https://v2.flowback.org/register" \
-d "username=flowback" \
-d "email=flowback@example.com"

    HTTP Request

    POST https://v2.flowback.org/register

    Body Parameters

    Parameter Notes
    username string
    email string

    Verify Register

    This endpoint completes an user registration, and creates a password associated to the user.

    curl "https://v2.flowback.org/register/verify" \
-d "verification_code=your_verification_code" \
-d "password=your_password"

    HTTP Request

    POST https://v2.flowback.org/register/verify

    Body Parameters

    Parameter Notes
    verification_code string Can be obtained here.
    password string
    Make sure that the user doesn't do any typing mistakes with the password by e.g. requiring the user to type their password twice.

    Forgot Password

    This endpoint resets the user's password. An activation email will be sent to the user containing the verification_code. If FLOWBACK_URL is specified in the backend .env, a link containing the verification_code as a query parameter will be sent instead. The code will be needed for the next step of the password reset.

    Example of the link: https://v2.flowback.org/forgot_password/

    If FLOWBACK_URL is specified in the backend's .env file, the user will instead receive a link:

    curl "https://v2.flowback.org/forgot_password" \
-d "email=flowback@example.com"

    HTTP Request

    POST https://v2.flowback.org/forgot_password

    Body Parameters

    Parameter Notes
    email string

    Verify Forgot Password

    This endpoint resets the user's password.

    curl "https://v2.flowback.org/forgot_password/verify" \
-d "verification_code=verification_code" \
-d "password=password"

    HTTP Request

    POST https://v2.flowback.org/forgot_password/verify

    Body Parameters

    Parameter Notes
    verification_code string Can be obtained here.
    password string
    Make sure that the user doesn't do any typing mistakes with the password by e.g. requiring the user to type their password twice.

    Notification

    Notification APIs span across multiple sections, however most notification management is handled within this section.

    Common Parameters

    Because of the confusion some parameters can give, this section will explain some common parameters.

    Channel

    Currently available sender_types: poll, group

    Each available sender_type got a section for Identity, Notification Categories, and Notification Subscribe

    For users to be able to trace back to the origin of their notification and be able to manage their subscriptions, the way this API achieve this is by having each notification object being sent through a channel. You can find sender types in the top of certain sections under the name Identity, you can also find categories under the name Notification Categories.

    Each channel can be identified using three parameters:

    Identifier Note
    sender_type str Source identifier type of the sender, e.g. group, user
    sender_id int ID of the sender_type, e.g. sent from group (sender_type) with id 1 (sender_id)
    category int Refers to a subsection for a sender_type, e.g. poll, schedule, delegate are categories within group

    Notification Object

    The notification object is the main content of each notification. Each notification that a user receives derives from a notification object.

    Common Parameters Note
    related_id int, optional The id related to the category of a channel, e.g. a group with category poll with id 3 (related_id)
    action str The action of a notification, the choices are create, update, delete
    message str The message for the notification
    timestamp str The time of the notification
    channel int The notification channel id related to this notification

    Notification

    The notification is what the user receives.

    Common Parameters Note
    user int The user receiving the notification
    notification_object int The notification object id related to this notification
    read bool Whether the user read the notification or not

    message__icontains char Message, but not case-sensitive
    action char Notification object action
    timestamp char Notification object timestamp
    timestamp__lt char Timestamp earlier than
    timestamp__gt char Timestamp later than
    channel_sender_type char Notification channel sender type
    channel_sender_id int Notification channel sender id
    channel_category char Notification channel category

    Output Parameters

    Parameter Notes
    id int The id of this notification
    read bool Whether the user read the notification or not
    object_id int The notification object id
    message char
    action char Notification object action
    timestamp char Notification object timestamp
    channel_sender_type char Notification channel sender type
    channel_sender_id int Notification channel sender id
    channel_category char Notification channel category
